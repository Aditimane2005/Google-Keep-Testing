# Test Plan – Google Keep Application

## 1. Objective
The objective of this testing is to verify the functionality of the Google Keep application and ensure all features are working as expected.

## 2. Scope
### Features to be tested:
- Create Note
- Edit Note
- Delete Note
- Pin/Unpin Note
- Add Reminder
- Change Note Color

### Features not in scope:
- Performance Testing
- Security Testing

## 3. Testing Type
- Manual Testing
- Functional Testing

## 4. Test Environment
- Device: Laptop / Mobile
- Browser: Chrome
- OS: Windows / Android

## 5. Test Data
- Sample text notes
- Reminder dates
- Different colors

## 6. Test Strategy
- Write test cases in TestRail
- Execute test cases
- Mark results as Pass/Fail
- Log defects for failed cases

## 7. Entry Criteria
- Application should be accessible
- Test cases should be ready

## 8. Exit Criteria
- All test cases executed
- Defects logged and documented

## 9. Deliverables
- Test Cases
- Test Execution Report
- Defect Report
- Screenshots

## 10. Risks
- Internet issues
- Application changes during testing
